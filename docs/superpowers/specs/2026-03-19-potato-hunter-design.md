# Potato Hunter 设计文档

> 以番茄工作法为核心驱动的游戏化效率工具。每一次专注都是一场狩猎，每一个任务都是一只独特的怪物。

## 1. 产品概念

### 一句话描述

怪物猎人式的番茄钟效率工具——用 GTD 管理任务，用番茄钟执行专注，用狩猎隐喻让重复的日常工作持续有成长感。

### 核心理念

- **专注 = 狩猎**：每个番茄钟是一次狩猎，完成即击杀
- **任务 = 猎物**：每个任务由 AI 生成为独特的怪物，有名字、属性、个性
- **成长 = 策略积累**：不是数值膨胀，而是用户学会了更好的搭配和应对方式
- **社区 = 同舟共济**：未来的协作 Boss 战，怪猎式联机体验（MVP 之后）

### 目标用户

需要日常专注工作的知识工作者、学生、创作者。单人可用，但社区感是核心价值壁垒。

### 平台

- MVP：Windows 桌面端（Tauri + React）
- 后续：macOS 桌面端

## 2. 核心循环

```
┌─────────┐    选择猎物     ┌─────────┐    完成番茄钟    ┌─────────┐
│  村 庄   │ ──────────→  │  狩 猎   │ ──────────→   │  结 算   │
│         │              │         │               │         │
│ · 任务清单│    准备出发    │ · 专注计时│    击杀/撤退    │ · 素材掉落│
│ · 时间规划│              │ · 最小浮窗│               │ · 经验记录│
│ · 装备搭配│              │ · 意图提示│               │ · 装备制作│
│         │   ←──────────│         │  ←──────────  │         │
└─────────┘    回村休息     └─────────┘    进入休息     └─────────┘
```

### 三个阶段

**村庄（非专注态）**
- 任务管理：GTD 式收件箱 → 整理 → 分配
- 时间规划：评估今天可用番茄数，分配给各任务
- 装备准备：查看/切换 build，为下一次狩猎做策略调整

**狩猎（番茄钟）**
- 最小浮窗显示任务意图 + 时间血条 + 基础控制
- 主界面最小化，不打扰用户
- 完成 = 击杀，中途放弃 = 撤退（详见撤退机制）

**结算（休息时间）**
- 短休息（5 分钟）：快速结算，微调下一轮计划
- 长休息（每 4 轮，15 分钟）：更丰富的回村体验，整理装备、回顾当天进度

## 3. 番茄钟微循环

番茄钟总时长可由装备（武器）配置，默认 25 分钟。内部分三个阶段：

```
│←── 准备 ──→│←──── 专注 ────→│←── 回顾 ──→│
│   固定2分    │   动态填充      │   固定3分   │
│ 确认目标     │   最小浮窗      │  记录成果    │
│ 制定策略     │   纯粹专注      │  反思收获    │
```

**计时规则：**
- 准备阶段：固定 2 分钟
- 回顾阶段：固定 3 分钟
- 专注阶段：= 总时长 - 准备 - 回顾（默认 25 - 2 - 3 = 20 分钟）
- 当装备改变总时长时（如 50 分钟武器），专注阶段自动伸缩（50 - 2 - 3 = 45 分钟），准备和回顾不变

### 准备阶段（2 分钟）

快速确认本次狩猎的目标和策略。倒计时开始，用于聚焦意图。准备阶段结束自动进入专注阶段。

### 专注阶段（~20 分钟）

主界面最小化，浮窗显示核心信息：

```
┌──────────────────────────────┐
│ 🎯 撰写需求文档第3部分        │
│ ▓▓▓▓▓▓▓▓▓▓▓░░░░░ 18:32     │
│ [暂停] [放弃] [详情]          │
└──────────────────────────────┘
```

进度条同时是倒计时和猎物血条——时间流逝 = 对怪物造成伤害。

### 回顾阶段（3 分钟）

专注阶段结束后自动进入回顾，剩余 3 分钟内完成：

- **「你完成了什么？」**（必填）—— 强制产出确认
- **反思引导**（选填，三选一降低认知成本）：
  - 很顺利，我做对了什么？
  - 遇到困难，问题是什么？下次如何改进？
  - 有新的想法或发现？

### 休息阶段

番茄钟结束后进入休息：

- 战利品展示动画
- 素材掉落结算
- 随机健康建议（起身走动、喝水、拉伸等）
- 下一个番茄钟预览（如果有的话）

## 4. 猎物系统

### 任务 → 猎物

用户创建任务时填写基础信息，AI 据此生成独特的猎物。

**用户填写：**
- 任务名称（必填）
- 任务描述（选填）
- 难度评估 + 预估番茄数（必填）
- 任务分类（工作/学习/创作/生活/其他）
- 重复设置

**AI 生成：**
- 怪物名称（和任务内容相关，有趣味性）
- 怪物属性（基于分类和难度）
- 视觉形象描述（MVP 阶段用 emoji/图标组合）

### 难度分级

| 难度 | 番茄数 | 猎物级别 | 行为 |
|------|--------|---------|------|
| 简单 | 1-2 | 小型猎物 | 单次或两次番茄钟击杀 |
| 中等 | 3-5 | 中型猎物 | 多次狩猎，每次削一段血 |
| 困难 | 6-10 | 大型猎物 | 建议拆分为子任务（部位破坏） |
| 史诗 | 11-20 | 巨型猎物 | AI 辅助拆分，必须分阶段攻略 |
| 传说 | >20 | Boss 级 | 未来多人协作目标 |

### 多番茄任务

中等以上的猎物需要多个番茄钟击杀：
- 猎物有血条，跨番茄钟持续追踪
- 每次回村能看到猎物当前剩余血量
- 最后一击完成时有特殊击杀结算

### 大任务拆分

预估 > 5 个番茄时，提示拆分（类似怪猎「部位破坏」）：
- AI 辅助将大任务拆成子任务
- 每个子任务 = 猎物的一个可破坏部位
- 破坏部位 = 额外素材掉落，鼓励拆分行为
- 提供三个选项：AI 帮我拆分 / 我自己拆分 / 坚持不拆分

## 5. 成长系统

### 核心原则

**你变强了，不是数值变大了。** 成长体现在拥有更多策略选择，更懂得针对不同任务搭配最优方案。

### 装备系统

**素材来源**：每次完成番茄钟，根据任务分类和完成质量随机掉落。

**装备类型（MVP 3 类）：**

| 装备位 | 功能隐喻 | 实际效果 |
|--------|---------|---------|
| 武器 | 专注方式 | 影响番茄钟节奏参数（如 25/5 vs 50/10） |
| 护甲 | 抗干扰策略 | 影响中断/暂停时的容错机制 |
| 道具 | 辅助工具 | 一次性使用，如「延长 5 分钟」「跳过准备阶段」 |

### Build 策略

不同任务类型适合不同装备搭配：
- 写作类 → 长时武器（50 分钟）+ 高容错护甲（允许短暂中断）
- 琐碎事务 → 短刀（15 分钟快速番茄）+ 连击道具（连续完成加成）
- 深度研究 → 重武器（长时专注）+ 回顾强化护甲（延长回顾阶段）

### 成长感来源

1. 解锁更多装备选项 → 策略空间变大
2. 积累对自己工作模式的认识 → 知道什么 build 适合什么任务
3. 这个认识是用户自己的成长，游戏只是让它可见

### 撤退机制

用户可以在狩猎中随时放弃（撤退）：

- **素材**：撤退不掉落任何素材（干净的「无奖励」，不是惩罚）
- **猎物血量**：撤退不对猎物造成伤害（本次番茄钟不计入击杀进度）
- **暂停**：暂停需要消耗一个道具（如「烟雾弹」），单次暂停上限 3 分钟，超时自动撤退。没有暂停道具时不能暂停，只能选择放弃（撤退）。这是有意的设计——专注期间中断不被鼓励。
- **撤退后流程**：跳过结算，直接返回村庄，显示简短提示「已安全撤退」
- **不扣任何东西**：撤退不扣素材、不扣装备耐久、不扣任何资源（暂停消耗的道具已在暂停时扣除）

### 装备效果定义

每种装备类型的 effect 字段使用结构化 JSON：

**武器（改变番茄钟节奏）：**
```json
{
  "type": "timer",
  "focus_minutes": 25,
  "break_minutes": 5,
  "long_break_minutes": 15,
  "rounds_before_long_break": 4
}
```

**护甲（改变容错机制）：**
```json
{
  "type": "tolerance",
  "max_pause_duration_seconds": 180,
  "allow_brief_interrupt": true,
  "brief_interrupt_seconds": 30
}
```

**道具（一次性消耗品）：**
```json
{
  "type": "consumable",
  "action": "extend_focus",
  "value": 5
}
```
可选 action：`extend_focus`（延长专注分钟数）、`skip_prep`（跳过准备阶段）、`bonus_loot`（本次掉落翻倍）。

### 制作系统

**位置**：村庄内的「工坊」区域。

**流程**：
1. 用户进入工坊，查看可制作的装备列表
2. 每件装备显示所需素材和当前持有量
3. 素材充足时，点击制作 → 确认消耗 → 获得装备
4. 新制作的装备自动加入装备库

**配方发现**：
- 基础装备配方从一开始就解锁
- 高级装备配方在首次获得某种稀有素材时解锁（「新配方发现！」提示）

**MVP 简化**：不做装备升级/强化，只做制作。装备只有「有」和「没有」，不分等级。

### 不做什么

- 不做等级系统、经验值、天赋树
- 不做数值膨胀（不会有「+15% 专注力」这种伪指标）
- 不做惩罚机制（撤退不扣任何东西）

## 6. 村庄系统

### GTD 任务管理

| GTD 概念 | 游戏语言 | 功能 |
|----------|---------|------|
| 收件箱 | 未确认情报 | 快速捕获任务，不需要立即评估 |
| 整理/分类 | 怪物鉴定 | 评估难度、分配番茄数、AI 生成猎物 |
| 待执行 | 猎物清单 | 已确认的待狩猎目标 |
| 今日计划 | 今日狩猎任务板 | 从清单中选入今天要打的猎物 |
| 已完成 | 狩猎图鉴 | 历史记录，击杀过的所有怪物 |

### 时间规划

**每日狩猎预算：**
- 用户设定今天可用的番茄钟数量
- 从猎物清单拖入今日任务板，系统自动累计所需番茄数
- 超出预算时提醒：「今天的猎物太多了，体力不够」
- 帮助用户做减法

**可视化示例：**
```
今日狩猎计划          [已分配 6/8 番茄]
──────────────────────────────────────
🐺 撰写需求文档      ▓▓▓░    3/4 番茄已完成
🦎 回复邮件          ░        0/1
🦎 整理会议纪要       ░        0/1
                     ── 剩余 2 番茄可分配 ──
```

### 狩猎图鉴

每只被击杀的怪物进入图鉴，记录：
- 怪物信息 + 实际花费番茄数 vs 预估
- 用户的完成笔记和反思
- 使用的装备搭配

隐性价值：用户逐渐积累对自己工作效率的认知——「写作类任务我总是低估，实际要多 1-2 个番茄」。

## 7. 随机性与新鲜感

### 设计原则

在不干扰工作的前提下，制造适度的变化感。随机事件只出现在村庄，绝不打断狩猎。

### 随机素材掉落

- 基础素材必定掉落（保底，不挫败）
- 稀有素材概率掉落（惊喜感）
- 任务分类影响素材池（写作类掉「墨水」系，编码类掉「齿轮」系）
- 连续完成多个番茄钟有额外掉落加成

### 猎物变体

- 同类型任务的怪物有不同外观、名字、性格描述
- 偶尔出现稀有变体（如「金色文书龙」），掉落更好的素材
- 让重复性任务不会每次面对完全相同的怪物

### 每日随机事件（第二阶段，MVP 不做）

未来可加入每天进入村庄时触发的小事件：
- 「旅行商人来了」—— 用素材交换稀有装备材料
- 「天气变化」—— 某类任务的掉落率微调
- 「村民请求」—— 完成额外小目标获得奖励

### 不做什么

- 不做需要用户复杂决策的随机事件
- 不做负面随机事件（不惩罚用户）
- 不做 gacha / 抽卡机制

## 8. 技术架构

### 整体结构

```
┌─────────────────────────────────────┐
│            Tauri v2 桌面壳           │
│  ┌───────────────────────────────┐  │
│  │        React 前端              │  │
│  │  ┌──────────┐ ┌──────────┐   │  │
│  │  │  村庄界面  │ │  浮窗组件  │   │  │
│  │  └──────────┘ └──────────┘   │  │
│  │  ┌──────────┐ ┌──────────┐   │  │
│  │  │  结算界面  │ │  图鉴界面  │   │  │
│  │  └──────────┘ └──────────┘   │  │
│  └───────────────────────────────┘  │
│                 ↕ Tauri Commands     │
│  ┌───────────────────────────────┐  │
│  │        Rust 后端               │  │
│  │  · 计时器（精准，不受前端影响）   │  │
│  │  · 本地数据存储（SQLite）        │  │
│  │  · 系统托盘 / 浮窗管理          │  │
│  │  · 通知管理                     │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
          ↕ HTTP（仅 AI 生成时）
┌─────────────────────────────────────┐
│         外部 AI API                  │
│   · 猎物生成（名称/描述/属性）        │
│   · 任务拆分建议                     │
└─────────────────────────────────────┘
```

### 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 桌面框架 | Tauri v2 | 轻量、安全、Rust 后端性能好 |
| 前端 | React + TypeScript | 生态成熟，组件库丰富 |
| 状态管理 | Zustand | 轻量，适合中小项目 |
| 样式 | Tailwind CSS | 快速迭代 |
| 本地存储 | SQLite（via Tauri 插件） | 结构化数据，支持复杂查询 |
| AI 调用 | Claude API | 猎物生成、任务拆分建议 |
| 计时器 | Rust 侧实现 | 不受前端渲染影响，精度可靠 |

### 关键技术决策

- **计时器在 Rust 侧**：前端 setInterval 在窗口最小化时不可靠，Rust 后端计时通过事件通知前端
- **浮窗 = Tauri 多窗口**：狩猎时创建 always-on-top 小窗口，主窗口隐藏
- **数据全部本地**：MVP 没有服务端，所有数据存本地 SQLite
- **AI 调用可选**：离线时可手动填写猎物信息，AI 只增强体验。具体回退方案见下方。

### AI 功能离线回退

| AI 功能 | 在线行为 | 离线/失败回退 |
|---------|---------|--------------|
| 猎物生成 | Claude API 生成名称/描述/属性 | 确定性名称生成器：任务分类 + 形容词池 + 名词池随机组合 |
| 任务拆分 | AI 分析任务并建议子任务 | 用户手动拆分（已有「我自己拆分」选项） |

API key 未配置或请求失败时，自动切换到回退方案，不阻断用户流程。

### 数据模型

```
Task（任务/猎物）
├── id, name, description
├── category（工作/学习/创作/生活/其他）
├── difficulty, estimated_pomodoros, actual_pomodoros
├── monster_name, monster_description, monster_variant
├── status（待鉴定/待狩猎/狩猎中/已击杀/已放弃）
├── total_hp, current_hp（血量 = estimated_pomodoros，每完成一个番茄钟减 1）
├── parent_task_id（拆分时的父任务关联）
├── repeat_config（不重复/每日/每周/自定义）
├── created_at, completed_at
└── tags[]（可选标签）

Material（素材）
├── id, name
├── category（墨水系/齿轮系/...，对应任务分类）
├── rarity（普通/稀有/传说）
└── icon

PlayerInventory（玩家库存）
├── materials: Map<material_id, quantity>
└── owned_equipment: equipment_id[]

LootDrop（掉落记录）
├── id, pomodoro_id
├── material_id
└── quantity

Pomodoro（番茄钟记录）
├── id, task_id
├── started_at, ended_at
├── result（完成/撤退）
├── completion_note（完成了什么）
├── reflection_type（顺利/困难/发现，可选）
├── reflection_text（反思内容，可选）
└── loadout_snapshot（本次使用的装备快照）

Equipment（装备模板）
├── id, name, type（武器/护甲/道具）
├── description
├── effect: JSON（见装备效果定义）
├── recipe: Map<material_id, quantity>
├── unlocked（配方是否已解锁）
└── is_consumable（道具为 true，武器护甲为 false）

Loadout（当前装备搭配）
├── weapon_id（可空）
├── armor_id（可空）
└── items: equipment_id[]（消耗品列表）

DailyPlan（每日计划）
├── date
├── total_budget（可用番茄数）
└── entries: PlannedTaskEntry[]

PlannedTaskEntry（每日计划条目）
├── task_id
├── planned_pomodoros_today（今天分配给这个任务的番茄数）
├── completed_pomodoros_today
└── order（执行顺序）
```

### 边缘情况处理

- **应用崩溃**：Pomodoro 记录在 started_at 时写入数据库。重启后检测到未完成的番茄钟，提示用户：已恢复上次的狩猎进度（基于已过时间恢复倒计时），或选择撤退
- **关闭浮窗**：等同于暂停，不等同于放弃。显示暂停提示，超时后自动撤退
- **计时器精度**：Rust 侧使用单调时钟（monotonic clock），不受系统时间调整影响
- **系统休眠/睡眠**：唤醒后比对单调时钟，如果休眠时间 < 剩余专注时间，恢复继续；否则自动撤退

### 通知设计（MVP）

| 触发时机 | 通知方式 |
|----------|---------|
| 准备阶段结束 | 浮窗内提示，无系统通知 |
| 专注阶段结束（进入回顾） | 浮窗内提示 + 系统通知（如果浮窗不在前台） |
| 番茄钟完成 | 系统通知 + 提示音（可选） |
| 休息结束 | 系统通知 + 提示音（可选） |
| 暂停超时即将撤退 | 系统通知警告 |

音效：MVP 阶段提供基础提示音（完成、休息结束），可在设置中开关。

## 9. MVP 范围

### 包含

| 模块 | 范围 |
|------|------|
| 村庄 | 收件箱 → 猎物鉴定 → 猎物清单 → 今日任务板 |
| 时间规划 | 每日番茄预算设定 + 任务分配 |
| 猎物生成 | AI 生成猎物名称/描述，离线时手动填写 |
| 番茄钟 | 准备（2分）→ 专注（动态，默认20分）→ 回顾（3分），总时长可由装备配置 |
| 浮窗 | 意图 + 时间血条 + 暂停/放弃/详情 |
| 结算 | 完成笔记（必填）+ 反思（选填）+ 素材掉落 |
| 休息 | 战利品展示 + 健康建议 + 下一轮预览 |
| 装备 | 3 类（武器/护甲/道具），基础制作系统 |
| 图鉴 | 已击杀猎物历史记录 |
| 平台 | Windows 桌面端 |

### 不包含

| 不做 | 原因 |
|------|------|
| 多人协作 / Boss 战 | 需要后端服务，第二阶段 |
| 社区功能 | 同上 |
| macOS 构建 | 先跑通 Windows |
| AI 图像生成 | 猎物视觉先用 emoji/图标 |
| 复杂随机事件 | 先做基础素材掉落随机性 |
| 数据同步 / 云端 | 纯本地存储 |
| 统计报表 | 图鉴够用 |
| 成就勋章系统 | 第二阶段高优先级 |

### 验证目标

MVP 成功标准：自己连续用一周，每天的番茄钟体验不无聊，且任务管理功能确实有用。

## 10. 未来规划

以下功能不在 MVP 范围内，按优先级排列：

### 第二阶段

- **成就勋章系统**：记录用户的里程碑（连续完成、首次击杀等），经历的证明而非数值奖励
- **协作 Boss 战**：怪猎式多人协作，每人贡献番茄钟 = 攻击回合，同舟共济
- **社区系统**：随机匹配组队、猎人公会
- **macOS 支持**

### 第三阶段

- **AI 图像生成**：为每只猎物生成独特视觉形象
- **详细统计报表**：专注时间分析、效率趋势
- **数据云同步**：跨设备使用
- **移动端**：作为补充使用场景
