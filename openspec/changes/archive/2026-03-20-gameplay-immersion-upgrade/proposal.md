## Why

MVP 完成后实际使用发现四个核心问题：视觉风格过于朴素缺乏游戏氛围，猎物生成的名称/描述没有代入感，多番茄怪物的体验存在 bug 且缺少持续追踪感，大怪物没有番茄组策略规划机制。

更根本的是，"打怪"的隐喻需要一个完整的世界观来支撑。新设定：**番茄农场被拖延怪物大军占领，你作为被召唤的勇者，要打败怪物、拯救番茄！**

> "每一件你拖延的事，都会召唤一只拖延怪物。每一次你选择专注，都是在斩杀怪物、夺回番茄。"

## What Changes

**世界观重构：召唤勇者 × 拖延怪物**
- 番茄农场是一片富饶的土地，番茄是世界的宝贵资源
- 拖延怪物是独立的敌对生物——每当任务被拖延，就会有怪物从虚空中被召唤，占领农场、囚禁番茄
- 任务越复杂，召唤的怪物越强大，囚禁的番茄越多
- 玩家是被农场召唤的勇者，每次专注 25 分钟 = 对怪物造成伤害
- 任务全部完成 = 怪物被击败，被囚禁的番茄获得自由
- 保留并强化战斗语言：击杀、HP、出击、撤退——这就是一场战斗

**视觉与沉浸感**
- 提升村庄和狩猎界面的视觉风格：像素风 + 怪猎崛起明媚调
- 结算画面：怪物倒下 → 番茄从牢笼中解放的胜利叙事
- 撤退文案：「怪物还在守着番茄，等你回来再战！」

**猎物生成增强**
- 更丰富的 AI prompt（融入世界观：拖延怪物占领农场）
- 更多样的离线名称池 + 属性标签（怪物的拖延类型）
- 怪物描述体现"这只怪物是什么样的拖延 → 它囚禁了几颗番茄"

**多番茄 bug 修复 + 番茄组机制**
- 修复 HP 追踪 bug
- 大怪物（>4 番茄）显示番茄组拆分（每组 ≤4，组间长休息）
- 每日计划增加排序和时间估算

**番茄收容所（原图鉴）**
- 图鉴改为「番茄收容所」——被拯救的番茄在此安家
- 每颗被救回的番茄有收藏展示
- 未完成的任务 = 番茄仍被囚禁（可视化动力）

## Capabilities

### New Capabilities

- `monster-attributes`: 属性标签系统（拖延类型标签），为每只怪物生成 1-3 个拖延属性
- `world-narrative`: 世界观叙事系统，统一所有 UI 文案、状态描述、结算画面的勇者战斗叙事

### Modified Capabilities

- `monster-system`: 融入拖延怪物叙事，增强生成质量（怪物名称/描述体现拖延特征），修复多番茄 HP 追踪 bug
- `village-task-management`: 番茄组显示、排序、预算可视化，文案适配战斗叙事
- `pomodoro-timer`: 浮窗视觉升级，番茄组进度显示，战斗主题视觉
- `hunt-journal`: 从"狩猎图鉴"转变为"番茄收容所"，展示被拯救的番茄集合

## Impact

- 部分文案更新（强化战斗叙事 + 拯救番茄主题）
- 前端组件大面积视觉和文案修改
- 结算流程增加"番茄解放"胜利叙事
- Rust monster_gen.rs 重构（拖延怪物世界观融入名称/描述生成）
- monster_cmd.rs AI prompt 重写（拖延怪物 prompt）
- Journal 组件改为番茄收容所
- 不涉及 schema 结构性变更（复用现有字段）
