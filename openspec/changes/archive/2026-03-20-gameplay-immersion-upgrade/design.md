## Context

Tomato Hunter MVP 已完成。核心循环可用但缺乏灵魂——缺少一个让人热血沸腾的世界观。

新的世界观：番茄农场被拖延怪物大军占领。每当任务被拖延，就会有怪物从虚空降临，占据农场、囚禁番茄。玩家是被召唤的勇者，通过专注战斗（番茄钟）击败怪物，解放被囚禁的番茄。

这是一场正面战斗——不是温柔的净化，而是热血的狩猎。保留并强化现有的战斗语言（击杀、HP、出击、撤退），加入召唤勇者和拯救番茄的叙事层。

## Goals / Non-Goals

**Goals:**
- 建立完整的"召唤勇者 × 拖延怪物 × 拯救番茄"世界观
- 采用像素风 + 怪猎崛起式明媚风格，打造温暖明亮但充满战斗感的世界
- 增强猎物生成质量和属性标签系统（拖延类型标签）
- 修复多番茄 HP 追踪 bug
- 引入番茄组和每日计划策略
- 将图鉴转变为番茄收容所

**Non-Goals:**
- 不做复杂动画（如粒子效果），用像素 sprite + 文字叙事
- 不改变游戏机制本身（装备、素材、计时器逻辑不变，强化包装语言）
- 不引入新的后端数据结构
- 不自制像素画（MVP 用开源像素资源 + emoji 过渡）

## Decisions

### D1: 叙事文案系统

**选择**: 建立中心化的叙事文案模块 `src/lib/narrative.ts`，管理世界观相关文案。

核心叙事元素：
```
世界观：番茄农场被拖延怪物占领，勇者被召唤来战斗
怪物：拖延怪物（独立敌对生物，不是番茄变的）
番茄：被怪物囚禁的宝贵资源
玩家：被召唤的勇者

保留的战斗语言（不替换）：
  击杀、HP、怪物、出击、撤退、猎物 — 这些本来就很好

增强/新增的文案：
  鉴定 → 侦查敌情
  未确认情报 → 未侦查的敌情
  猎物清单 → 讨伐清单
  图鉴 → 番茄收容所
  结算 → 胜利结算（怪物倒下，番茄获救）
  放弃 → 撤退（怪物太强了，暂时撤退！）
```

**理由**: 保留现有战斗语言比全部替换成"净化"更自然、更热血。只需要增强叙事层（召唤、拖延怪物、拯救番茄），不需要大面积术语替换。

### D2: 视觉风格 — 像素风 + 怪猎崛起明媚调

**选择**: 整体视觉走「明媚像素风」路线，参考怪猎崛起的和风明亮调性：
- 村庄底色用温暖的米黄/浅木色调（类似怪猎炎火村）
- 点缀色用翠绿（自然）、樱粉（生机）、天蓝（晴空）
- 怪物区域用暗紫/灰雾色，与明亮村庄形成反差——击败怪物 = 让颜色回来
- 像素字体用于标题和关键数字（如计时器、番茄计数）
- 卡片、按钮带轻微像素风边框和阴影（CSS 实现，不需要图片资源）

**替代方案**: 暗黑哥特风、扁平极简风
**理由**: 明媚风配合拯救叙事——农场本来是明亮的，怪物入侵带来了阴暗。每次击杀怪物都是让一片区域恢复光明。像素风在 Steam 上有成熟受众，且开发/视觉一致性成本极低。

**像素资源策略**：
- MVP 阶段：用 CSS 像素化效果（`image-rendering: pixelated`、像素风边框）+ 像素字体 + emoji 作为怪物占位
- 后续：替换为自制或委托的像素 sprite sheet（每分类 5-8 种怪物形态 + 获救番茄形态）

### D3: 像素字体选型

**选择**: 使用开源像素字体：
- 中文标题/关键 UI：[Zpix (最像素)](https://github.com/SolidZORO/zpix-pixel-font) — 开源免费，支持中文像素字体
- 数字/计时器：等宽像素风，CSS `font-family: monospace` + letter-spacing 模拟即可
- 正文/描述：保持系统默认字体，不像素化（保证可读性）

**理由**: 标题和关键数字用像素字体建立风格感，正文保持可读性。不强求全局像素化。

### D4: 结算叙事 — 胜利解放

**选择**: 击败怪物后的结算画面采用胜利叙事风格：
```
⚔️ 怪物被击败了！

「拖延怪物倒下了...」
「一颗被囚禁的番茄重获自由！」
🍅 拯救了一颗番茄！

战利品：
⚙️ 齿轮零件 x2
🔮 通用碎片 x1
```
部分击败（多番茄任务中完成一轮）：
```
「怪物受到重创，但还没倒下！」
「你隐约看到被囚禁的番茄在笼中闪烁...」
⚔️ 继续战斗！
```
撤退：
```
「怪物太强了，暂时撤退！」
「被囚禁的番茄还在等着你回来...」
```

**理由**: 热血战斗风的结算比温柔净化更有成就感和驱动力。

### D5: 番茄收容所 — 复用 Journal 组件

**选择**: Journal 组件重命名为 TomatoSanctuary，但数据结构不变。每只被击败的怪物对应一颗获救的番茄。收容所中显示：番茄 + 它被哪只怪物囚禁的故事 + 战斗记录。番茄墙读取「净化番茄」素材数量。

**理由**: 纯前端改造，不涉及后端。被拯救的番茄构成成就感来源。

### D6: 拖延可视化 — 囚禁时间指示

**选择**: 任务创建时间距今越长，卡片的色彩饱和度越低、边框越灰暗（像素风中表现为颜色褪色）。这让用户直觉感受到"番茄被囚禁太久了，在逐渐枯萎"。
**替代方案**: 暗紫渐变
**理由**: 在明媚调性下，"失去颜色"比"变暗"更有紧迫感——被囚禁越久，番茄越虚弱。

### D7: 世界观呈现 — 召唤勇者设定

**选择**: 在村庄首页增加世界观入口，首次进入时展示简短的召唤故事：
```
🍅 番茄农场告急！

拖延怪物大军从虚空降临，
占领了这片富饶的番茄农场。
番茄们被囚禁，光芒逐渐黯淡...

你，被农场最后的力量召唤而来。
拿起武器，击败怪物，拯救番茄！

⚔️ 开始战斗
```
之后可通过 lore 按钮重新查看。

**理由**: 轻量的世界观呈现，不阻塞使用流程，但给新用户足够的代入感。

## Risks / Trade-offs

**[像素字体的中文支持]** → Zpix 覆盖了常用汉字但不是全部。缓解：只在标题和关键 UI 用像素字体，正文用系统字体。

**[明媚风格和战斗主题的视觉平衡]** → 整体明亮但怪物区域需要暗色反差。缓解：只在怪物卡片和狩猎浮窗中使用暗色调，村庄主体保持明媚。

**[像素资源长期维护]** → 后续需要定制 sprite sheet。缓解：MVP 用 emoji + CSS 像素效果过渡，不阻塞上线。

**[战斗叙事可能太中二]** → 保持克制，文案有力但不过度。拖延怪物的名字可以有趣但不低幼。
