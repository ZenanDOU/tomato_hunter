## Context

Tomato Hunter 当前的装备系统是简单数值修改器：武器改时长参数，护甲改暂停容忍度，消耗品通过素材合成。番茄收容所是纯展示。整个经济体系缺乏深度——没有货币、没有定价、没有被动产出。

现有数据库中的装备种子数据（标准短刀、長弓、匕首、皮甲、重甲、烟雾弹、时光沙漏、猎人直觉）需要全部替换为新设计。

已完成的音频系统（SynthEngine + SfxManager + MmlPlayer）可以直接支持护甲的音频功能（白噪音、报时音效）。

## Goals / Non-Goals

**Goals:**
- 武器成为"节奏策略"选择，每种有独特的 flow 逻辑（不只是数值差异）
- 护甲成为"沉浸辅助"选择，控制专注阶段的音频体验
- 建立番茄农场被动经济系统，驱动消耗品兑换循环
- 设计可持续的数值平衡（产出与消耗匹配）
- 消耗品提供有意义的临场决策空间

**Non-Goals:**
- 不实现弱点洞察消耗品（需要与策略/任务清单系统联动，另行设计）
- 不实现急救包（当前无即时素材获取判定）
- 不做装备升级/强化系统
- 不做装备耐久度/损耗机制

## Decisions

### D1: 武器作为 Timer Mode 而非参数修改器

**选择**: 每种武器对应一个独立的 Timer Mode（sword/dagger/hammer），Rust 后端 timer 引擎按 mode 执行不同 flow 逻辑。

**替代方案**:
- A) 继续用 JSON 参数修改 → 无法表达匕首的循环选择和重锤的半程判定
- B) 前端逻辑控制 → 计时精度不够，且状态管理复杂

**理由**: 匕首的"暂停选择行动/休息"和重锤的"25分钟半程奖励"都是 flow 级别的差异，不是简单参数能描述的。Timer Mode 在 Rust 侧实现保证了计时精度和状态一致性。

**三种 Mode 的 Flow**:

```
Sword (默认):
  prep(2min) → focus(20min) → review(3min) → [break(5min) → next round]
  连续4轮后 long_break(15min)

Dagger:
  [选择阶段(暂停)] → focus(15min) → [选择阶段(暂停)] → focus/rest(15min) → ...
  无自动休息，用户每轮选择"行动"或"休息"
  番茄计数 = ceil(action_count / 2)

Hammer:
  prep(3min) → focus(44min) → review(3min) = 50min total
  25min后撤退 → 1 tomato
  完成 → 2 tomatoes
  不可连续（完成后强制返回村庄）
```

### D2: 护甲控制专注阶段音频

**选择**: 护甲 effect 中新增 `audioMode` 字段（silent / white-noise / interval-alert），AudioManager 在进入 focus 阶段时根据当前护甲切换音频行为。

**理由**: 已有的音频系统（SynthEngine 的 noise 波形 + SfxManager 的定时触发）完全可以支持。白噪音 = 持续的 noise buffer 播放，报时 = 每 180 秒触发一次 timer SFX。

**与栖息地 BGM 的关系**: 护甲音频优先于栖息地 BGM。进入 focus 阶段时，护甲接管音频控制：
- 棉甲: 停止 BGM，静音
- 皮甲: 停止 BGM，播放白噪音
- 重甲: 停止 BGM，每 3 分钟播放报时音效

非 focus 阶段（prep、review、break、village）仍由栖息地/场景 BGM 控制。

### D3: 番茄农场经济模型

**选择**: 番茄素（Tomato Essence）作为通用货币，产出公式与专注时长和农场番茄数量绑定。

**产出公式**:
```
每分钟产出 = floor(农场番茄数 / 10) + 1
```
- 0-9 颗番茄: 每分钟 1 番茄素
- 10-19 颗番茄: 每分钟 2 番茄素
- 20-29 颗番茄: 每分钟 3 番茄素
- 以此类推

只在专注阶段（focus phase）产出。prep/review/break 不产出。

**每日重置**: 番茄素余额不重置，持续积累。农场番茄数 = 累计解救番茄数（历史总量）。

### D4: 消耗品定价体系

**设计原则**:
- 基础消耗品便宜，频繁使用
- 强力消耗品贵，慎重使用
- 一次25分钟标准番茄约产出 20-25 番茄素（以10颗农场番茄、每分钟2素计，20分钟focus = 40素；以0-9颗算 = 20素）

**定价表**:

| 消耗品 | 效果 | 价格(番茄素) | 设计意图 |
|--------|------|-------------|---------|
| 烟雾弹（允许暂停） | 暂停3分钟 | 15 | 基础保险，约1次番茄产出的 60-75% |
| 时间延伸 | 延长专注+3分钟 | 10 | 便宜，鼓励坚持 |
| 休息延伸 | 延长休息+2分钟 | 8 | 最便宜，照顾休息需求 |
| 时间压缩 | 缩短本次专注-5分钟 | 20 | 较贵，惩罚提前退出但比撤退好 |
| 策略跳过 | 跳过策略制定阶段 | 12 | 中等，偶尔用 |
| 复盘跳过 | 跳过复盘阶段 | 12 | 中等，但建议少用 |
| 双倍掉落 | 下次狩猎素材翻倍 | 30 | 贵，约1.5次番茄产出 |
| 番茄肥料 | 农场产出翻倍1小时 | 25 | 贵，长期投资型 |

### D5: 合成配方与素材映射

**设计**: 每种武器/护甲有唯一配方，使用对应栖息地的素材。默认装备（手剑+棉甲）免费拥有，不需合成。

| 装备 | 所需素材 | 栖息地关联 |
|------|---------|-----------|
| 匕首 | 齿轮零件×5 + 精密齿轮×1 | 齿轮工坊(work) |
| 重锤 | 知识结晶×8 + 智慧宝石×2 | 遗忘图书馆(study) |
| 皮甲 | 生活纤维×5 + 生命露珠×1 | 荒废花园(life) |
| 重甲 | 墨水碎片×5 + 灵感精华×1 + 通用碎片×3 | 枯竭画廊(creative) + 通用 |

### D6: 数据库迁移策略

新增一个迁移文件 `005_workshop_farm_rework.sql`:
- 替换 equipment 种子数据（新武器/护甲/消耗品定义）
- 新增 `tomato_farm` 表（农场状态、番茄素余额）
- equipment 表新增 `price` 字段（消耗品的番茄素价格）
- equipment 表 `effect` JSON 结构更新（weapon → timer_mode, armor → audio_mode）
- 旧存档迁移：旧武器 → 手剑，旧护甲 → 棉甲，旧消耗品 → 等量烟雾弹

## Risks / Trade-offs

**[匕首循环模式的 Rust 实现复杂度]** → 匕首需要 timer 引擎支持"暂停等待用户选择"状态，不同于现有的自动推进逻辑。需要新增 `awaiting_choice` phase。

**[经济平衡难以一次做对]** → 初始数值基于估算，实际使用后可能需要调整。将定价放在数据库中而非硬编码，便于后续调整。

**[破坏性迁移]** → 所有现有装备数据将被替换。需要处理已有玩家存档。迁移脚本将旧装备映射到新装备（最接近的等价物）。

**[护甲音频与现有音频系统的交互]** → 护甲接管 focus 阶段音频后，AudioManager.enterHabitat() 的 BGM 切换逻辑需要条件化——只在非 focus 阶段生效。需要明确的优先级规则。

## Open Questions

- 番茄肥料"产出翻倍1小时"的计时器在用户不专注时是否倒计时？建议：只在专注时消耗肥料时间。
- 匕首模式下"选择休息"算不算番茄计数？建议：不算，只统计选择"行动"的次数。
