## Context

番茄猎人是一款游戏化番茄钟应用（Tauri + React + Zustand + SQLite）。当前讨伐列表（HuntList）展示 status 为 `ready` 或 `hunting` 的顶级任务，但没有提供清理机制。随着使用时间增长，过期任务堆积。UI 已有 `captive-fade` 样式暗示任务老化（1/3/7 天逐级变淡），但无法对老化任务采取行动。

新手引导方面，当前仅有一个一次性 lore modal（localStorage 存储），没有功能性引导。

技术栈：React + Zustand + TailwindCSS + SQLite（Tauri plugin-sql），像素风格设计系统。

## Goals / Non-Goals

**Goals:**
- 让用户可以从讨伐列表释放不需要的怪物，保持列表整洁
- 释放操作保留历史数据（软删除），不丢失任务记录
- 支持批量释放以提高效率
- 提供交互式新手引导帮助新用户上手

**Non-Goals:**
- 不做"回收站"或撤销释放功能（released 状态是终态）
- 不做自动清理/过期机制（由用户主动决定）
- 不做完整教程系统，仅做轻量引导
- 不迁移现有 localStorage lore 标记（保持简单）

## Decisions

### D1: 释放而非删除

**选择**: 新增 `released` 状态，而非使用现有 `deleteTask`（硬删除）。

**理由**: 保留历史数据用于统计（如"总共面对过多少怪物"）。且叙事上"释放怪物回归野外"比"删除任务"更契合猎人世界观。`deleteTask` 方法保留不变，`released` 是一个新的终态，与 `killed` 并列。

**替代方案**: 硬删除 — 简单但丢失历史数据，且与 `abandoned` 状态语义不清。

### D2: 释放入口 — 展开详情后的操作按钮

**选择**: 在 HuntItem 展开详情面板中添加"释放"按钮，与"出击"按钮并列。对于拆分怪物，释放整只怪物（parent + 所有 children）。

**理由**: 隐藏在展开面板中避免误触，且与现有交互模式（点击展开查看详情和操作）一致。不需要额外的批量选择 UI。

### D3: 批量释放 — 长按或专用模式

**选择**: 在列表顶部添加"清理"按钮，进入批量选择模式，用户勾选要释放的怪物后确认。

**理由**: 当积累了大量过期怪物时，逐个释放太慢。批量模式是标准的列表管理模式。进入批量模式时，每个怪物卡片左侧显示勾选框。

### D4: 引导方式 — Spotlight + Tooltip

**选择**: 半透明遮罩 + 镂空高亮 + 像素风格 tooltip 逐步引导。

**理由**: 让用户在真实 UI 上学习，沉浸感好。遮罩阻止用户点击非目标区域，确保引导流程顺畅。

### D5: 引导状态 — settings 表

**选择**: 引导完成/进度存入 SQLite settings 表。

**理由**: 与现有 settings 模式一致，比 localStorage 更可靠。

## Risks / Trade-offs

- **[释放拆分怪物]** 释放 parent 时需级联释放所有 children → 在 `releaseTask` 中处理级联，一个事务内完成
- **[已加入今日计划的怪物]** 释放后需从今日计划中移除对应条目 → `releaseTask` 同时清理 daily_plan_entries
- **[正在狩猎中的怪物]** 不允许释放 status=hunting 的怪物 → UI 层禁用释放按钮
- **[引导步骤元素定位]** tab 切换时目标元素可能不在 DOM 中 → 引导流程在切换 tab 后等待目标元素出现再定位
