## Context

Tomato Hunter 的怪物系统经过两轮迭代，已有 25 个种族和属性标签系统。但生态系统感不足，用户反馈怪物像随便拼凑的，标签令人困惑，拆分部位不合理。同时怪物仍使用 emoji 占位，需要建立像素画生成管线。

## Goals / Non-Goals

**Goals:**
- 设计生态学合理的怪物体系（栖息地+食物链，非进化链）
- 每个种族有独特的身体结构和拆分部位
- 用清晰的种族特性替代抽象属性标签
- 讨伐清单支持查看任务详情
- 建立像素画生成管线，为每个种族生成实际的像素图

**Non-Goals:**
- 不增加怪物属性克制等复杂战斗系统
- 不做复杂的 sprite sheet 动画（静态像素图即可）

## Decisions

### D1: 怪物生态系统 — 栖息地 × 食物链

**选择**: 每个任务分类对应一个栖息地，每个栖息地有 3 个不同物种（小型猎物→中型捕食者→顶级掠食者），它们是同一生态环境中共存的不同物种，不是进化关系。

| 栖息地 | 分类 | 生态描述 | 小型猎物 (simple/medium) | 中型捕食者 (hard/epic) | 顶级掠食者 (legendary) |
|--------|------|----------|------------------------|----------------------|---------------------|
| 齿轮工坊废墟 | work | 废弃的机械工坊，锈蚀的齿轮和管道间栖息着以金属为食的生物 | 齿轮虫 ⚙️ — 啃食锈铁的小型甲虫 | 铁甲蜈蚣 🐛 — 在管道中穿行的节肢捕食者 | 锻炉蟒 🐍 — 盘踞在废炉中的顶级掠食者 |
| 枯竭画廊 | creative | 被遗弃的画廊，干涸的颜料和褪色的画布间飘荡着以创意能量为食的生物 | 墨点虫 🖋️ — 吸食干涸墨水的微小虫群 | 灵感蛾 🦋 — 被创意之光吸引的夜行蛾 | 画境龙 🐲 — 从废弃画作中苏醒的古老存在 |
| 遗忘图书馆 | study | 无人问津的图书馆深处，书页间滋生着以知识为食的生物 | 书虱 📖 — 啃食书页的微小寄生虫 | 论文狼 🐺 — 在书架间游猎的群居捕食者 | 知识龙 🐉 — 盘踞在禁书区的远古守护者 |
| 荒废花园 | life | 无人打理的花园，杂草丛生的角落里藏匿着被自然遗忘的生物 | 杂草鼠 🌿 — 在杂草间筑巢的小型啮齿类 | 藤蔓蛙 🐸 — 用藤蔓捕猎的伏击型两栖类 | 古树熊 🐻 — 与最古老的树融为一体的巨型兽 |
| 迷雾沼泽 | other | 常年笼罩迷雾的沼泽地，能见度极低，栖息着虚实不定的生物 | 迷雾虫 👻 — 在雾气中聚散的发光虫群 | 虚空鸦 🦇 — 穿行于雾层间的暗影飞行者 | 混沌龙 🐲 — 迷雾的源头，混沌的化身 |

**食物链逻辑**: 小型物种在环境中觅食（虫→吃锈铁/墨水/书页/杂草/雾气）→ 中型物种捕食小型（蜈蚣吃虫、蛾吃虫、狼吃虱、蛙吃鼠、鸦吃虫）→ 顶级掠食者统治整个栖息地（蟒、龙、龙、熊、龙）。

**理由**: 食物链是真实的生态关系，蜈蚣吃甲虫、蟒蛇吃蜈蚣——这在自然界完全合理。比"蜂进化成蟒"可信得多。

| 画境龙 | 🌈 幻象·自我怀疑 / 🔥 焚稿·推倒重来 | 反复否定已有成果 |

### D4: 讨伐清单详情面板

**选择**: 点击怪物卡片 accordion 展开详情。显示：任务完整信息、怪物故事、栖息地/食物链位置、种族特性、部位结构预览、出击按钮。

### D5: 像素画生成管线

**选择**: 使用 Canvas API 程序化生成像素画。每个种族定义一个像素矩阵（16×16 或 32×32），存储在 bestiary 数据中。前端用一个 `PixelSprite` 组件将矩阵渲染为 Canvas。

```typescript
// 种族像素画定义
interface SpriteData {
  width: number;   // 16 or 32
  height: number;
  pixels: string;  // 压缩的颜色数据，如 "0011100..." 映射到调色板
  palette: string[]; // ["transparent", "#EE4433", "#333333", ...]
}
```

像素画来源（优先级）：
1. 手动用代码定义像素矩阵（15 个种族，每个 16×16 = 256 像素，完全可控）
2. 后续可导入外部像素画工具（如 Aseprite）导出的 JSON sprite data

`PixelSprite` 组件：
- 接收 SpriteData，用 Canvas 逐像素绘制
- 支持缩放（×2, ×3, ×4），使用 nearest neighbor 保持像素清晰
- 替代现有的 emoji `.monster-sprite`

**替代方案**:
- Claude API 动态生成（不可控，风格不一致）
- 外部像素资源库（版权风险，风格不匹配）
**理由**: 代码定义像素矩阵是最可控的方式。16×16 的像素画只有 256 个像素点，可以直接在代码中定义。风格统一、加载快、无版权问题。

## Risks / Trade-offs

**[种族减少到 15 个]** → 每个质量大幅提升。后续可添加同栖息地的变种。

**[手绘 15 个像素画]** → 每个 16×16 = 256 像素，工作量可控。可以先做 5 个代表性的，其余逐步补充。

**[旧数据兼容]** → parseMonsterVariant 保持向后兼容。
