## Context

村庄主界面（VillageLayout）当前有 6 个 Tab：侦察、讨伐、计划、工坊、农场、档案。侦察（Inbox）处理未鉴定任务，讨伐（HuntList）展示已鉴定的待战/狩猎中怪物。两者是同一任务生命周期的不同阶段，拆分为两个 Tab 增加了导航成本。

农场（TomatoFarm）当前包含统计数据和战斗历史，但战斗历史与农场主题不相关。农场本身只有被动展示，缺乏互动感。

## Goals / Non-Goals

**Goals:**
- 将 Tab 数从 6 个减少到 5 个，降低导航复杂度
- 合并侦察和讨伐为统一的「狩猎看板」，让任务流转更直观
- 将战斗历史迁移到猎人档案，与统计/图鉴形成完整的猎人记录
- 为农场增加浇水和施肥互动，提升经营参与感

**Non-Goals:**
- 不改变 DailyPlanBoard、Workshop 的功能或结构
- 不改变后端数据库 schema 或 Tauri commands
- 不改变怪物生成、分裂、狩猎等核心战斗机制
- 不改变番茄素的产出公式或经济平衡

## Decisions

### D1: 合并策略 — 状态分组看板

**决策**：创建 `HuntBoard` 组件替代 Inbox + HuntList，按任务状态分 3 个折叠区段：
1. 📥 未鉴定（unidentified）— 原 Inbox 功能
2. ⚔️ 就绪/战斗中（ready + hunting）— 原 HuntList 功能
3. 🏷️ 快速录入表单 — 保持在顶部

**理由**：状态是任务流转的自然分界线，用户可以在同一个界面完成「录入→鉴定→查看就绪」的完整流程。折叠区段让信息密度可控。

**替代方案**：Tab 内嵌子 Tab — 但这只是把问题下移一层，增加了嵌套复杂度。

### D2: 战斗历史迁移 — 猎人档案新增区块

**决策**：在 HunterProfile 中新增「战斗日志」折叠区段，位于图鉴下方。数据源不变（已击杀任务），只是展示位置从 TomatoFarm 移至 HunterProfile。

**理由**：战斗记录是猎人成就的一部分，与统计/图鉴放在一起形成完整档案。农场可以专注于经营玩法。

### D3: 农场互动 — 浇水/施肥机制

**决策**：
- **浇水**：免费操作，冷却时间 30 分钟（真实时间）。浇水后农场所有番茄获得「润泽」状态，下一个番茄钟的番茄素产出 +50%。
- **施肥**：消耗 1 个「番茄肥料」道具（现有 consumable ID 15）。施肥效果与现有 fertilizer 逻辑一致（产出翻倍 60 分钟专注时间），但增加视觉反馈。
- **视觉**：农场画面增加土壤/植株生长状态指示，浇水后显示水滴动画，施肥后显示植株发光效果。

**理由**：浇水提供免费但有限的互动循环，施肥复用现有经济系统。两者叠加但不相乘，保持经济平衡。

### D4: Tab 结构调整

**决策**：新的 5 个 Tab 顺序：
1. 📋 计划（Plan）— 保持默认 Tab
2. ⚔️ 狩猎（HuntBoard）— 合并后的看板
3. 🔨 工坊（Workshop）— 不变
4. 🌱 农场（Farm）— 增强互动
5. 📜 档案（Profile）— 新增战斗日志

**理由**：计划是每日最高频操作保持首位。狩猎放第二方便从计划跳转。农场和档案是低频浏览，放后面。

## Risks / Trade-offs

- **[合并看板信息过载]** → 使用折叠区段 + 数量徽标，默认只展开有内容的区段
- **[浇水冷却状态管理]** → 存储在 farmStore 内存状态即可，不需要持久化（刷新后重置冷却是可接受的）
- **[战斗历史迁移后农场显得空]** → 增加农场视觉丰富度和互动来弥补
