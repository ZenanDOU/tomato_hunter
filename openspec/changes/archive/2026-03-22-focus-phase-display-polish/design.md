## Context

FocusPhase 是一个 always-on-top 浮动窗口，用户专注工作时全程可见。当前布局：
- Row 1: 大时钟（96px pixel-title）
- Row 2: 任务名称（左）+ 控制按钮（右：策略图标、暂停、撤退）
- Row 3: 策略笔记（可折叠，默认隐藏）

窗口默认高度 176px，展开策略时 240px。

道具系统中，专注阶段可用的消耗品：
- ID 7 烟雾弹：暂停（已实现，只检查有无但不显示数量）
- ID 8 时间延伸：+3分钟
- ID 10 时间压缩：-5分钟

当前 `useConsumable` 和 timer 的 `extendFocus` / `shortenFocus` 可能需要在 timerStore 中确认是否已有对应方法。

## Goals / Non-Goals

**Goals:**
- 任务名称作为视觉主体，居中显示
- 怪物名作为装饰性元素，低调呈现
- 策略笔记默认展示（有内容时）
- 添加道具快捷栏，显示可用道具数量，支持一键使用

**Non-Goals:**
- 不改变计时器逻辑
- 不改变道具消耗逻辑（useConsumable 已存在）
- 不重新设计整个 hunt 窗口

## Decisions

### D1: 紧凑布局 — 时钟 → 图标+任务名+策略 → 道具栏

**选择**: 三行紧凑布局：
1. 大时钟（保持不变）
2. 怪物像素图标 + 任务名居中 + 策略笔记（同区域，任务名大字居中，策略小字在下方，怪物图标在任务名左侧作为装饰）
3. 底部道具快捷栏 + 撤退按钮

**理由**: 怪物不需要文字名称，像素图标已经足够辨识且是装饰性元素。任务名是核心信息，居中显示。策略笔记紧跟其下，整体只需三行。

### D2: 道具快捷栏 — 仅显示拥有的可用道具

**选择**: 底部水平排列，每个道具显示图标+数量，点击直接使用。只显示数量>0且当前阶段可用的道具。

**理由**: 不显示没有的道具避免信息噪音。点击即用，无需额外确认（道具效果都是正面的，且有数量限制）。

### D3: 窗口高度 — 固定高度适应新布局

**选择**: 固定窗口高度为约 220px，不再动态切换。策略笔记超长时 truncate 为单行。

**理由**: 动态高度变化会分散注意力。固定高度更稳定。

## Risks / Trade-offs

- **[道具 action 对接]** 需要确认 timerStore 是否有 extendFocus/shortenFocus 方法，如果没有需要先实现 → 检查后决定
- **[窗口高度]** 新布局信息更多，需要平衡信息密度和窗口紧凑 → 控制在 220px 内
