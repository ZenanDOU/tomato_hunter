## Context

当前工坊 UI 中，EquipmentSection 以扁平卡片列出所有装备，每个卡片显示名称 + 完整描述 + 操作按钮。描述文本过长（手剑描述 ~50 字，包含完整计时规则），在紧凑卡片中信息过载。

怪物系统已有成熟的 32x32 贴图体系（spriteData32.ts + PixelSprite 组件），可复用。

## Goals / Non-Goals

**Goals:**
- 6 件装备各有独立的 32x32 像素贴图
- 卡片默认视图：贴图 + 名称 + 一句话核心效果
- 点击卡片展开：完整规则说明 + 素材需求
- 消耗品也精简为名称 + 价格

**Non-Goals:**
- 不为装备添加动画帧（静态贴图即可）
- 不修改装备数据库模型
- 不添加新装备
- 不修改装备效果逻辑

## Decisions

### D1: 贴图数据格式

复用 spriteData32.ts 的 `SpriteData32` 格式（palette + frames），但只需一个 idle 帧（无动画）。

装备贴图独立存放在 `src/lib/equipmentSprites.ts`，按 equipment ID 索引。

设计方向：
- **手剑**：简单直剑，金属灰 + 棕色握柄
- **匕首**：短刃双刃，银色 + 暗色柄
- **重锤**：大锤头，铁灰 + 木柄
- **棉甲**：白色软甲轮廓
- **皮甲**：棕色皮质纹理
- **重甲**：铁灰金属板甲

### D2: 卡片 UI 重构

**折叠态**（默认）：
```
┌─────────────────────────────┐
│ [贴图] 手剑        ⚔️装备中 │
│         25 分钟标准节奏      │
└─────────────────────────────┘
```

**展开态**（点击后）：
```
┌─────────────────────────────┐
│ [贴图] 手剑        ⚔️装备中 │
│         25 分钟标准节奏      │
│─────────────────────────────│
│ 2 分钟策略 + 20 分钟专注     │
│ + 3 分钟复盘                 │
│ 连续完成 4 轮后 15 分钟长休息 │
│                    [装备]    │
└─────────────────────────────┘
```

操作按钮（装备/制作）移入展开面板，折叠态只显示状态标签。

### D3: 核心效果一句话

每件装备提供一个 `shortDesc` 字符串，显示在折叠态：

| 装备 | shortDesc |
|---|---|
| 手剑 | 25 分钟标准节奏 |
| 匕首 | 15 分钟快攻循环 |
| 重锤 | 50 分钟深度专注 |
| 棉甲 | 安静模式 |
| 皮甲 | 白噪音背景 |
| 重甲 | 定时提醒 |

这些不存数据库，而是在前端代码中维护一个 `EQUIPMENT_SHORT_DESC` 常量映射（equipment ID → 短描述）。

### D4: PixelSprite 静态模式

PixelSprite 组件已支持无动画渲染（legacy 格式就是静态的）。新格式只提供一个 idle 帧也可以正常渲染。不需要修改 PixelSprite。

## Risks / Trade-offs

- **[贴图质量]** 32x32 手写像素贴图对审美要求较高 → 使用简单轮廓 + 分类色系（武器暖色、护甲冷色）保证风格统一
- **[展开/折叠状态]** 多个卡片可同时展开 → 允许多个展开，不强制手风琴模式，降低复杂度
