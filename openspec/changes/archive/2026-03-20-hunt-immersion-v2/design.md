## Context

Tomato Hunter 已完成世界观升级（拖延怪物叙事 + 像素风视觉），但实际使用暴露了体验断层：怪物生成只是一个按钮点击、大任务无法拆分、准备阶段无互动、怪物名与任务脱节、狩猎中看不到真正要做的事。

当前技术栈：React 19 + Tauri 2 + SQLite + Rust 后端。前端用 Zustand 状态管理 + Tailwind CSS v4。

## Goals / Non-Goals

**Goals:**
- 建立内置怪物种族库，为每个分类×难度提供具象的怪物形象
- 侦查敌情后展示"怪物发现卡片"，给予视觉和叙事反馈
- 提供身体部位拆分机制，让大任务自然分解为可执行的子任务
- 准备阶段从被动倒计时改为主动策略确认
- 狩猎浮窗中同时展示怪物信息和真实任务信息

**Non-Goals:**
- 不做 AI 像素图生成（后续迭代）
- 不做自定义怪物图片上传
- 不引入复杂的战斗系统或属性克制
- 不改变现有的计时器机制和核心游戏循环

## Decisions

### D1: 怪物种族库数据结构

**选择**: 创建 `src/lib/bestiary.ts` 存储内置怪物种族库。每个种族有：id、名称后缀、分类、难度范围、emoji 图标（MVP）、像素图路径（后续）、描述模板。

```typescript
interface MonsterSpecies {
  id: string;           // e.g. "work-gear-beast"
  suffix: string;       // e.g. "齿轮兽"
  category: TaskCategory;
  difficulties: TaskDifficulty[];
  emoji: string;        // MVP 用 emoji
  pixelArt?: string;    // 后续替换为像素图路径
  descTemplates: string[]; // "这只{adj}{suffix}从你拖延的{task}中诞生..."
}
```

每个分类预设 5 个种族，覆盖不同难度：
- work: 齿轮兽、报表蛙、会议蟒、邮件鸦、加班狼
- study: 书卷龙、考试虎、笔记蝶、公式蛇、论文狼
- creative: 墨灵、灵感蝶、画布蛇、脑洞猫、设计鹰
- life: 绿藤怪、收纳蛙、采购鹿、打扫蜂、做饭熊
- other: 影魔、混沌虫、迷雾鸦、拖延龙、虚空鱼

**理由**: 固定种族库可以预制像素图资源，后续每个种族一张图即可覆盖。比完全随机生成更有收集感。

### D2: 关联性命名 — 任务关键词 + 种族后缀

**选择**: 怪物名称由"任务相关形容词 + 种族后缀"组成。

关联性生成逻辑：
1. 从任务名称提取关键词（取前 2-4 个字）
2. 加上拖延主题形容词
3. 匹配分类下的一个种族后缀

例如：任务"写季度报告" → "拖延的·报告蟒"
例如：任务"背单词" → "走神的·单词龙"

AI 在线模式下，仍然让 AI 生成更有创意的名字，但基于种族库的 suffix 提示。

**替代方案**: 纯随机名称（当前做法）
**理由**: 任务关键词入名让用户一眼知道这个怪物代表什么任务，增强代入感。

### D3: 侦查反馈 — 怪物发现卡片

**选择**: 侦查敌情后，不是直接跳转，而是弹出一个"怪物发现卡片"模态框：
```
┌──────────────────────────┐
│    🐺 (大号 emoji/像素图)   │
│                          │
│  「拖延的·报告蟒」          │
│   ⚔️ 中等 · 4 HP          │
│   🏷️ 拖延 · 焦虑           │
│                          │
│  "这只报告蟒从你迟迟不动笔  │
│   的季度报告中诞生..."      │
│                          │
│      [加入讨伐清单]         │
└──────────────────────────┘
```

**理由**: 给"侦查"这个动作一个仪式感，让用户感受到怪物的具象存在。

### D4: 身体部位拆分

**选择**: 大怪物（建议 >4 番茄）可以拆分为身体部位。固定提供 3 个部位槽位：

| 部位 | 隐喻 | 含义 |
|------|------|------|
| 🧠 头部 | 思考/规划 | 任务中需要规划、构思的部分 |
| 💪 身体 | 执行/主体 | 任务中主要的执行工作 |
| 🦶 脚部 | 收尾/交付 | 任务中的收尾、检查、提交工作 |

拆分后：
- 每个部位成为一个子任务（parent_task_id 指向原任务）
- 用户必须填写每个部位的名称和描述（具体工作内容）
- 用户分配每个部位的番茄数（总和 = 原任务番茄数）
- 原任务变为"容器"状态，不直接狩猎
- 子任务各自独立完成（击杀子任务 = 打掉一个部位）

**理由**: 身体部位隐喻直观，倒逼用户将模糊的大任务拆解为"规划→执行→收尾"。

### D5: 准备阶段互动式改造

**选择**: PrepPhase 从 2 分钟倒计时改为互动式确认流程：

1. **确认出击**（主画面）：展示怪物信息 + 任务信息 + "确认讨伐？"
2. **策略思考**（可选）：提示"准备从哪里开始入手？"，提供文本框让用户记录策略
3. **点击"开始战斗"后** → 自动进入 focus 阶段

准备阶段不再是纯倒计时，而是一个用户主动确认+思考的步骤。策略笔记存入 pomodoro 记录的新字段 `strategy_note`。

窗口尺寸在准备阶段使用较大的尺寸，进入 focus 后缩回小窗。

**替代方案**: 保持倒计时 + 加一个可选输入框
**理由**: 番茄钟的核心理念是"先想清楚再做"，准备阶段应该主动引导这个过程而非被动等待。

### D6: 狩猎浮窗信息增强

**选择**: FocusPhase 浮窗增加信息层：
```
┌──────────────────────────┐
│  🎯 写季度报告              │ ← 真实任务名称
│  ⚔️ 拖延的·报告蟒 · 💪身体  │ ← 怪物名 + 当前部位（如有拆分）
│  ████████░░  18:32         │ ← 进度条 + 时间
│  番茄 2/4                   │ ← 总进度
│              [暂停] [撤退]  │
└──────────────────────────┘
```

**理由**: 用户在专注时需要看到"我在做什么"而不只是"我在打什么怪物"。

### D7: 像素图资源策略

**选择**: MVP 阶段使用放大的 emoji + CSS `image-rendering: pixelated` 效果模拟像素风。每个种族对应一个 emoji。后续替换为实际像素图 sprite。

种族 emoji 映射在 bestiary.ts 中定义，UI 用 `.monster-sprite` class 显示（大号 emoji + 像素化渲染）。

**理由**: 不阻塞开发，视觉效果可接受，后续只需替换图片路径。

## Risks / Trade-offs

**[拆分功能增加复杂度]** → 子任务通过 parent_task_id 关联，完成逻辑需要处理"所有子任务完成 = 父任务完成"。缓解：子任务独立管理，父任务仅作为容器。

**[准备阶段去掉倒计时可能太随意]** → 用户可能在准备阶段停留太久。缓解：增加一个柔和的时间提示（"已准备 5 分钟"），但不强制。

**[关联性命名可能生成奇怪的名字]** → "提取关键词"逻辑可能不准。缓解：fallback 到种族库的默认形容词。

**[emoji 像素化效果在不同平台表现不一]** → Tauri 使用 webview 渲染，emoji 样式取决于系统。缓解：MVP 可接受，后续换真实像素图。
