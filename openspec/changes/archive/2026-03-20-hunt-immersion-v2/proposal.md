## Why

上一轮沉浸感升级后实际使用发现五个核心体验问题：怪物生成没有视觉反馈（点一下按钮就完了），大任务缺少拆分机制，准备阶段缺乏策略互动，怪物名称与任务脱节缺少代入感，狩猎过程中看不到真实任务信息。

## What Changes

**怪物生成视觉反馈**
- 侦查敌情后展示"怪物卡片"：内置像素图 + 故事性描述 + 属性标签
- 内置怪物种族库：每个分类（work/study/creative/life/other）× 难度预设 3-5 种怪物形象，每种有固定像素图
- 后续迭代支持 AI 动态生成像素图

**怪物拆分（身体部位隐喻）**
- 大怪物（>4 番茄）提供拆分功能，拆分为"头部/身体/四肢"等部位
- 每个部位是一个子任务，用户必须填写该部位对应的具体工作内容和描述
- 拆分后的部位在 UI 中显示为怪物身体图解，完成一部分 = 打掉一个部位

**准备阶段互动式改造**
- 准备阶段窗口放大，视觉风格与村庄一致（明媚像素风）
- 增加互动确认："确认讨伐这只怪物吗？"
- 策略提示：引导用户思考"准备从哪里开始入手？"
- 可选策略笔记：记录本次出击的计划/思路，存入 pomodoro 记录

**怪物与任务关联性增强**
- 怪物名称包含任务关键词（如"报告"任务 → "报告蟒"而非随机"齿轮兽"）
- 内置怪物种族库按分类匹配，名称由"任务关键词 + 种族后缀"组成
- AI 在线模式下生成与任务内容高度相关的名称和描述

**狩猎浮窗信息增强**
- 显示真实任务名称（而非仅怪物名）
- 显示预计进度（第 N/M 番茄）
- 如有拆分，显示当前攻击的部位名称和对应的具体任务

## Capabilities

### New Capabilities

- `monster-bestiary`: 内置怪物种族库系统，包含每个分类×难度的怪物种族定义、像素图资源引用、命名规则
- `monster-splitting`: 怪物身体部位拆分系统，大任务拆分为头部/身体/四肢等子任务，含部位图解 UI
- `prep-phase-interactive`: 准备阶段互动式改造，含确认对话、策略提示、策略笔记功能

### Modified Capabilities

- `monster-system`: 怪物生成增加侦查反馈卡片展示、关联性命名、种族库匹配
- `pomodoro-timer`: 狩猎浮窗增加真实任务信息、进度显示、当前部位提示

## Impact

- 新增 `src/lib/bestiary.ts` 怪物种族库数据
- 新增像素图资源（`public/monsters/` 目录，初期可用 emoji + CSS 像素化过渡）
- 修改 Inbox 组件：侦查后展示怪物卡片
- 新增 MonsterSplitForm 组件：身体部位拆分 UI
- 修改 PrepPhase 组件：放大窗口 + 互动确认 + 策略笔记
- 修改 FocusPhase 组件：显示任务信息和进度
- 修改 Rust monster_gen.rs：关联性命名逻辑
- 数据库 pomodoros 表增加 strategy_note 字段
