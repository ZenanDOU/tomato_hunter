## Context

Tomato Hunter 是番茄列车系列的姊妹产品。番茄列车已有完整的 8-bit 像素风设计规范（色板、字体、UI 组件、动画），Tomato Hunter 需要对齐这套视觉语言，同时保留"狩猎怪物"的独特氛围。

当前技术栈：React 19 + Tailwind CSS v4 + Tauri 2。样式通过 `@theme` 定义 CSS 变量，组件使用 Tailwind utility classes。

## Goals / Non-Goals

**Goals:**
- 建立与番茄列车一致的色板和 CSS 变量
- 创建可复用的像素风 UI 组件（按钮、卡片、进度条）
- 统一村庄和狩猎窗口的视觉风格
- 提升像素风纯度（纯色填充、像素描边、整数缩放）

**Non-Goals:**
- 不制作自定义像素图资源（角色、场景 tile 等）
- 不实现视差滚动背景或复杂动画
- 不引入音效系统
- 不改变游戏机制或数据结构

## Decisions

### D1: 色板映射 — 番茄列车 → 番茄猎人

**选择**: 直接采用番茄列车核心色板，加入猎人特色的战斗/怪物色。

| 番茄列车色 | 用途（猎人） |
|-----------|------------|
| 天空蓝 #55BBEE | 村庄背景、准备阶段背景 |
| 草地绿 #5BBF47 | 成功状态、完成指示、薄荷按钮 |
| 番茄红 #EE4433 | 出击按钮、HP 条、怪物强调 |
| 阳光黄 #FFD93D | 奖励、星星、番茄计数高亮 |
| 云朵白 #FFFFFF | 卡片背景、文字底 |
| 像素黑 #333333 | 描边、正文、轮廓 |
| 暖橙 #FF8844 | CTA 按钮、能量条 |
| 米白 #FFF8E8 | 内容区域背景 |
| 深蓝 #3366AA | 专注阶段背景、链接色 |
| 浅粉 #FFCCDD | 装饰、柔和强调 |

战斗/怪物专用：
| 色 | 用途 |
|---|------|
| 暗红 #AA2222 | 怪物 HP 损失、危险状态 |
| 暗紫 #443355 | 怪物卡片背景（保留但优化） |

**理由**: 6-3-1 原则 — 天空蓝+云朵白占 60%（背景），暖橙+番茄红占 10%（强调），其余为辅助。

### D2: 像素描边系统

**选择**: 用 CSS `outline` + `box-shadow` 模拟像素描边和阴影，替代当前的 inset box-shadow。

```css
.pixel-box {
  outline: 2px solid #333333;
  outline-offset: -2px;
  box-shadow: 2px 2px 0 0 rgba(0,0,0,0.2);
}
```

**替代方案**: SVG 边框、border-image
**理由**: 纯 CSS 实现，无额外资源依赖，视觉效果接近像素描边。

### D3: 设计系统组件

**选择**: 创建 3 个基础像素风组件：

1. **PixelButton**: 五态按钮（normal/hover/pressed/disabled/cta），对齐番茄列车规范
2. **PixelCard**: 带像素描边和右下阴影的卡片容器
3. **PixelProgressBar**: 分段纯色填充进度条

全部使用 Tailwind utility classes + 自定义 CSS class，不引入新依赖。

**理由**: 这三个组件覆盖了 80% 的 UI 表面积，统一它们就能显著提升一致性。

### D4: 狩猎窗口风格方案

**选择**: 按阶段使用不同色调，但保持在番茄列车色板范围内：

| 阶段 | 背景 | 主色 | 氛围 |
|------|------|------|------|
| 准备 | 天空蓝 #55BBEE 渐变 | 暖橙 | 明亮，鼓舞斗志 |
| 专注 | 深蓝 #3366AA | 番茄红进度条 | 沉稳，专注 |
| 复盘 | 米白 #FFF8E8 | 草地绿 | 温暖，成就感 |
| 结算 | 云朵白 #FFFFFF | 阳光黄 | 欢快，奖励 |

注意：不使用渐变填充（番茄列车规范禁止），而是用分区纯色填充或微妙的 CSS 分段色。

**替代方案**: 全部使用深色主题
**理由**: 深色主题与番茄列车的明亮欢快基调不符。不同阶段用不同色调既保持统一又有节奏感。

### D5: 像素字体策略

**选择**: 保留 Zpix 作为中文像素标题字体。正文使用系统字体。与番茄列车规范中"H1/H2 像素字体 + Body 系统字体"的策略一致。

### D6: 组件替换策略

**选择**: 渐进替换。先创建设计系统组件，然后逐个替换各页面中的原始 Tailwind 样式。不一次性重写所有组件，避免引入大量 diff。

替换顺序：
1. CSS 主题变量（影响全局）
2. PixelButton（替换所有 button 元素）
3. PixelCard（替换所有卡片容器）
4. PixelProgressBar（替换 HP 条和进度条）
5. 狩猎窗口各阶段配色

## Risks / Trade-offs

**[色板差异大]** → 从暗色调跳到明亮高饱和需要仔细调试对比度。缓解：遵循 4.5:1 对比度规则。

**[像素风与 Tailwind 的冲突]** → Tailwind 的 rounded、shadow 等 utility 不符合严格像素规范。缓解：用自定义 CSS class 覆盖，逐步迁移。

**[狩猎窗口明亮风格可能分散注意力]** → 专注阶段用深蓝底色保持低刺激，只有 PrepPhase 和 ReviewPhase 使用明亮色。
